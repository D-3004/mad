Experiment No: 10
Title: Google Lighthouse PWA Analysis

Aim:
To test the performance and PWA functionality of a website using Google Lighthouse tool.

Requirements:

* Google Chrome Browser

---

### Step 1: Open Website

Open Google Chrome

Open any website (example):

https://www.amazon.in

---

### Step 2: Open Developer Tools

Press:

Ctrl + Shift + I

---

### Step 3: Open Lighthouse

Click on:
👉 Lighthouse tab

(If not visible → click ">>" and select Lighthouse)

---

### Step 4: Configure Audit

Select all categories:

* Performance ✔
* Accessibility ✔
* Best Practices ✔
* SEO ✔
* PWA ✔

---

### Step 5: Run Audit

Click:
👉 "Analyze page load" / "Run audit"

Wait for 30–60 seconds

---

### Step 6: View Report

Lighthouse will generate a report showing scores such as:

* Performance Score
* Accessibility Score
* Best Practices
* SEO
* PWA

---

### Output:

A Lighthouse report is generated displaying performance, accessibility, SEO, and PWA analysis of the website.

---

### Conclusion:

Google Lighthouse tool is successfully used to analyze and test the website performance and PWA functionality.




Description:

In this experiment, the performance and Progressive Web App (PWA) characteristics of a website were analyzed using the Google Lighthouse feature available in Google Chrome. The objective was to evaluate the quality, performance, and best practices of a web application through automated auditing.

The experiment involved selecting a live website (such as Amazon India) and running a Lighthouse audit via the browser’s Developer Tools. Multiple audit categories were selected, including Performance, Accessibility, Best Practices, SEO, and PWA, allowing for a comprehensive evaluation of the website.

Lighthouse analyzes various parameters such as page load speed, responsiveness, use of modern web standards, accessibility compliance, and search engine optimization. It also checks for PWA-related features like service workers, installability, and offline capabilities. Based on these checks, it generates a detailed report with scores and recommendations for improvement.

This experiment demonstrates how developers can use automated tools to identify performance bottlenecks, improve user experience, and ensure compliance with modern web development standards.




Conclusion (Improved):

In this experiment, the Google Lighthouse tool was successfully used to analyze the performance and PWA capabilities of a website. The audit provided detailed insights into various aspects such as speed, accessibility, SEO, and adherence to best practices.

The key learning outcome is understanding how automated auditing tools can help evaluate and optimize web applications for better performance and user experience. It also highlights the importance of PWA standards in modern web development.

Overall, this experiment provides practical knowledge of testing and improving website quality using industry-standard tools like Lighthouse, which is essential for building efficient and user-friendly web applications.
